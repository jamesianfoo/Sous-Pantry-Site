01

Brand Principles

Cooking is craft.
So is every screen.

Sous Pantry sits on the kitchen counter, splattered with olive oil, used in mid-recipe panic. It needs to feel calm, capable, and a little bit beautiful. These principles guide every design decision.

Warm, not sterile.

We borrow from cookbooks and farmer's markets, not SaaS dashboards. Beige paper, hand-set type, soft shadows. Avoid pure white. Avoid gunmetal grey. The app should feel like something you'd happily leave open on the bench.

Information, then ornament.

Pantry counts, expiry dates, missing ingredients — these are the job. Decorative type and warm tones serve the data, never compete with it. Numbers are tabular. Quantities are prominent. Every flourish earns its place.

Confidently local.

Coles, Woolies, Aldi, IGA. Metric. Aussie ingredient names. We don't pretend to be from somewhere else. The product is opinionated about being made here, and the design reflects that with subtle warmth rather than loud flag-waving.

Premium without precious.

Polished, not pretentious. We use editorial typography and considered spacing — but we're still the app you'll use with one hand while stirring. Touch targets stay generous. Copy stays plain. Beauty bows to function when they conflict.

Do

Pair Beige with Navy at high contrast. Reserve Forest for Pro and success states. Use Gold only for premium tier moments and milestone celebrations. Allow generous Beige to breathe — it's the most distinctive thing about us.

Don't

Don't use pure white (#FFFFFF) — it breaks the warmth. Don't add new accent hues. Don't use Terracotta for primary actions; Navy is the action button. Don't apply gradients to anything except the paywall and Pro CTA — gradients are reserved.

Built on a 4-point grid.

10

Motion

Movement that cooks slowly.

Sous Pantry doesn't bounce, slide, or flash. Motion eases gently in and decisively out — like a souffle rising, not a notification popping. Standard curve is a soft cubic-bezier; durations stay under 320ms unless something genuinely deserves more.

Snap

120ms · ease-out
Taps, toggles, hovers.

Standard

240ms · cubic-bezier(0.2, 0.8, 0.2, 1)
Sheets, tabs, transitions.

Considered

320ms · ease-in-out
Modals, hero reveals.

Celebration

520ms + spring
Milestone unlock, streak save.

Rules of Restraint

One animated thing at a time, ideally. Never animate text content (only its appearance). Always honour prefers-reduced-motion: drop durations to 0ms and skip springs. Save spring physics for milestone celebrations only — they're the loudest tool we have, so use them when it matters.

11

Voice & Tone

Like a friend who actually cooks.

Warm. Direct. Occasionally a bit cheeky. We're not a wellness brand, we're not a productivity tool, and we're definitely not your grandmother's recipe blog. We talk like a confident home cook — no jargon, no hedging, no "synergy".

Sounds like us

"You've got 8 of 11 ingredients. Grab parsley and feta — dinner's sorted."

"Make it mine. Tweak this recipe and save your version."

"Streak saved. That feta was getting close."

"Cherry tomatoes expire tomorrow. Here are 3 things to make tonight."

Doesn't sound like us

"Empower your culinary journey with AI-driven insights."

"Oops! Looks like something went wrong 😅"

"Unlock the power of premium meal planning today!"

"We hope you're having a great day! Just a friendly reminder…"

Tone Spectrum

Empty states & success: warm.
Errors & expiry alerts: direct.
Paywall & Pro CTAs: confident.
Educational tooltips: plain.

Australian English

Colour, flavour, organise. Capsicum not bell pepper. Coriander not cilantro. We say "chips" for hot, "crisps" only for British brands.

Don't be cute

No "Whoops!", no exclamation marks in errors, no emoji in body copy (light use in playful empty states only). Skip "Just" before verbs — it weakens us.

Code Tokens

Copy this, ship faster.

Drop these straight into your SwiftUI, Compose, or web codebases. Names match the design system 1:1, so a designer saying "use space-5 with shadow-2" maps directly to SP.space5 / SPSpacing.s5 / --space-5.

SwiftUI

// Sous Pantry — Theme.swift (actual implementation)
import SwiftUI

enum AppTheme {
    // MARK: - Colours
    static let navy      = Color(red: 0.086, green: 0.141, blue: 0.216)  // #162437
    static let green     = Color(red: 0.176, green: 0.353, blue: 0.239)  // #2D5A3D
    static let cream     = Color(red: 0.969, green: 0.953, blue: 0.925)  // #F7F3EC
    static let softMint  = Color(red: 0.890, green: 0.929, blue: 0.902)  // #E3EDE6
    static let gold      = Color(red: 0.769, green: 0.588, blue: 0.353)  // #C4965A
    static let slate     = Color(red: 0.392, green: 0.439, blue: 0.502)  // #647080

    // MARK: - Gradients
    static let navyGradient = LinearGradient(
        colors: [navy, Color(red: 0.118, green: 0.196, blue: 0.298)],
        startPoint: .topLeading, endPoint: .bottomTrailing
    )
    static let greenGradient = LinearGradient(
        colors: [green, Color(red: 0.224, green: 0.435, blue: 0.298)],
        startPoint: .topLeading, endPoint: .bottomTrailing
    )
}

// MARK: - View helpers
extension View {
    func soCard(padding: CGFloat = 16) -> some View {
        self
            .padding(padding)
            .background(Color(.systemBackground), in: RoundedRectangle(cornerRadius: 14))
            .shadow(color: AppTheme.navy.opacity(0.07), radius: 8, x: 0, y: 2)
    }

    func sectionLabel(_ text: String) -> some View {
        VStack(alignment: .leading, spacing: 12) {
            Text(text)
                .font(.caption.weight(.semibold))
                .tracking(1.2)
                .foregroundStyle(AppTheme.slate)
                .textCase(.uppercase)
            self
        }
    }
}
